What is Meta Event Match Quality (EMQ)?

Meta Event Match Quality (EMQ) is a score from 0 to 10 measuring how accurately the customer data your Conversions API sends matches Meta's user records. When you send a purchase event, Meta tries to match it to a specific person using signals like email address, phone number, IP address, and browser fingerprint. The more signals match, the higher your EMQ score — and the better Meta's algorithm can optimise your campaigns.

Most stores using only the browser pixel score between 3.5 and 5.5. Stores with Shopify's native CAPI typically see 5.0–6.5. True server-side CAPI implementations consistently reach 7.5–9.0.

EMQ score bands and what they mean

0–6.0Poor — algorithm flying blind, CPM inflated
6.0–7.0Acceptable — gaps in iOS or pixel blocking
7.0–8.0Good — effective optimisation
8.0–10Excellent — peak Advantage+ performance

What is a good EMQ score for Meta?
A score of 6.0 is the bare minimum for Advantage+ Shopping Campaigns to function effectively. 7.0–8.0 is considered good. 8.0+ is excellent and required for maximum algorithmic performance.
Why does Shopify's native CAPI give a low EMQ?
Shopify's built-in CAPI is still browser-triggered — the thank-you page pixel must fire before the server event sends. If iOS or an ad blocker blocks that page, the server event never fires at all.
How much ad spend is wasted with a low EMQ?
Studies suggest 25–40% of Meta ad spend is wasted through poor signal quality, as Meta's algorithm optimises against incomplete conversion data.
